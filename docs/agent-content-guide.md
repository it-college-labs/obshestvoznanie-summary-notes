# Инструкция для агентов: конспекты Нейроархива

Эта инструкция нужна для агентов, которые добавляют статьи, аннотации,
изображения и конфиги. Интерфейс уже строится из реестра статей, поэтому
добавление нового материала не должно требовать правок в сценах, стилях или
роутинге.

## Контентная единица

Одна тема курса равна одной карточке недели и одному MDX-файлу. Каждая статья
добавляется в `src/content/articles.ts` и должна заполнить поля:

```ts
{
  id: "week-03-politics",
  week: "Неделя 3",
  title: "Политика и власть",
  annotation: "Короткая живая аннотация на 1-2 строки.",
  tags: ["политика", "власть", "государство"],
  accent: ["#7dd3fc", "#a78bfa", "#fb7185"],
  folderPreviewImages: [imageOne, imageTwo, imageThree],
  botThinkingImage,
  mdx: PoliticsArticle,
  readingTime: "9 минут",
  status: "ready",
}
```

`id` должен быть стабильным, латиницей и без пробелов. Для GitHub Pages прямые
ссылки идут через hash route: `#/article/week-03-politics`.

## Как писать MDX

Не копируй один и тот же план в каждую статью. Структура зависит от темы:
где-то нужна таблица, где-то цепочка причин, где-то серия примеров. Постоянным
должен быть уровень ясности, а не шаблон.

Обязательный минимум качества:

- дай понятные определения ключевых терминов через `<Definition />`;
- покажи хотя бы один жизненный или учебный пример через `<Example />`;
- добавь одну сильную мысль через `<KeyIdea />`;
- предупреди о типичной ошибке через `<ExamTrap />`, если в теме есть риск
  путаницы;
- между вставками обязательно оставляй обычный объясняющий текст: простые
  Markdown-абзацы или `<Text>...</Text>` / `<PlainText>...</PlainText>`;
- используй `<CompareTable />` только там, где сравнение реально помогает.

Статья не должна быть набором одних плашек. Вставки нужны для акцентов:
определений, примеров, ловушек, ключевых мыслей и таблиц. Основная логика
темы должна раскрываться обычным текстом между ними. Хороший ритм: 2-4
абзаца объяснения, затем одна смысловая вставка, затем снова обычный текст.

```mdx
<Text>
  Общество удобно понимать не как толпу людей, а как устойчивую систему
  связей. Эти связи держатся на ролях, нормах, институтах и ожиданиях.
</Text>

<Definition term="Социальный институт">
  Устойчивая форма организации совместной жизни: семья, школа, государство,
  право, религия.
</Definition>

Обычный Markdown-абзац тоже поддерживается и подходит для продолжения мысли
без отдельной визуальной плашки.
```

Длина статьи: обычно 700-1200 слов. Не раздувай текст ради объема. Важнее,
чтобы преподаватель видел цельность и аккуратную работу с понятиями.

## Визуальные данные

Для каждой темы нужны 3 вертикальных превью. Они показываются при наведении на
папку, поэтому должны различаться по смыслу: например, термин, схема, ситуация.
Технические требования:

- вертикальный формат примерно 260x360 или близкий;
- без мелкого нечитаемого текста;
- не использовать одно и то же изображение три раза;
- если настоящих ассетов еще нет, временно используй placeholder из
  `public/assets/placeholders`.

`accent` задает три цвета темы. Подбирай палитру по смыслу темы, но не делай
все статьи одним оттенком. Цвета должны хорошо работать на светлой карточке.

## Аннотация и теги

Аннотация должна звучать как вход в тему, а не как сухой пункт учебника. Хорошо:
“Как общество удерживает порядок: нормы, санкции, мораль, право и тонкая
граница между привычкой и правилом.” Плохо: “В данной статье рассматриваются
социальные нормы.”

Теги короткие, 2-4 штуки. Они помогают быстро понять, что внутри папки.

## Проверка перед сдачей

После добавления статей нужно выполнить:

```bash
npm run lint
npm run build
```

Потом вручную открыть сайт, проверить папку новой недели, hover/tap превью и
саму статью. Если статья не собирается, чаще всего причина в JSX внутри MDX:
незакрытый тег, массив без кавычек или лишний символ в пропсах.
